# 白地図マインスイーパ (Hakuchizu Minesweeper)

白地図マインスイーパは、日本地図や世界地図などの「実際の地理データ（GeoJSON）」を盤面に見立てた新感覚のマインスイーパゲームです。
通常の四角いグリッドとは異なり、都道府県や国々の「実際の隣接関係」に基づいて地雷の数を推測するため、遊びながら自然と地理・地形を学べる知育＆パズルアプリとなっています。

## 💡 なぜ開発をしたのか（動機・背景）

1. **「パズル」×「地理学習」の融合**:
   従来のマインスイーパは規則的なマス目ですが、これを「現実の地図」に置き換えたらどうなるか？という好奇心からスタートしました。北海道のように巨大で隣接が少ないエリアもあれば、長野県のように8つもの県と隣接するエリアもあり、地域ごとの特性がそのままゲームの難易度や戦略に直結する面白さを追求しました。
2. **直感的な学習体験の提供**:
   ただ暗記するだけの地理学習ではなく、マインスイーパのロジックを解き進める過程で「ここが開いたから、隣の〇〇県には地雷がある！」と、能動的に位置関係を把握できるツールを目指しました（マスを開放するとプチ雑学が表示されるエンタメ要素も取り入れています）。
3. **モダンWeb技術への挑戦**:
   SVGによる複雑なベクターマップの描画、レスポンシブなUI、スマートフォンでの直感的なピンチズームやドラッグ操作など、モダンなフロントエンド技術を総合的に活用するポートフォリオとして最適だと考えました。

| --- | --- |
| **React (TypeScript)** | 複雑なゲーム状態（各エリアの開閉、フラグ、地雷配置など）を宣言的に管理するため。また、GeoJSONのような複雑なデータ構造を扱うにあたり、TypeScriptの型安全性がバグの早期発見に不可欠でした。 |

| **Tailwind CSS** | クラスベースで直感的にスタイリングができ、開発スピードを大幅に向上させるため。特に「ダークモード（`dark:`）」や「レスポンシブ対応（`md:`）」の記述が容易で、洗練されたUIを素早く構築するのに役立ちました。 |
| **Vercel** | React/Viteとの親和性が非常に高く、GitHubにプッシュするだけで自動的にビルド＆グローバルなエッジネットワークへデプロイされるCI/CD環境をゼロ設定で構築できるため。 |

## 🔥 苦労した点と、それをどう乗り越えたか

### 1. 不規則な「隣接関係」の計算と探索ロジック
* **課題**: 通常のマインスイーパは周囲8マスを調べれば済みますが、地図の場合は「どの県とどの県が隣接しているか」が不定形です。
* **解決策**: 事前にGeoJSONの座標データから各境界線を解析し、正確な「隣接リスト（Adjacency List）」をJSONとして事前生成・キャッシュする仕組みを構築しました。また、空白マス（周囲に地雷がないマス）を一気に開放する処理には、再帰的な探索（再帰/キューを用いた幅優先探索）を実装し、市区町村モードのような数百の要素がある盤面でもブラウザがフリーズしないようパフォーマンスを最適化しました。

### 2. スマホ・PC両対応の快適なマップ操作体験（UX）
* **課題**: マップの「タップ（マスを開く）」「ロングタップ（フラグを立てる）」「ドラッグ（マップを移動）」「ピンチ（拡大縮小）」という複数の操作が競合し、スマホ実機で意図しない誤爆（スワイプ中にマスが開いてしまう等）が多発しました。
* **解決策**: タッチイベントとD3のイベントを綿密に分離しました。
  * `d3-zoom` の開始時に座標を記録し、一定ピクセル以上指が動いた場合は `isDragging` フラグを立ててクリックイベントをキャンセルするロジックを実装。
  * ロングタップ判定タイマー実装時には、`event.touches.length >= 2`（2本指でのタッチ）を検知した瞬間にタイマーを破棄することで、ピンチズーム中に誤ってフラグが立つ不具合を完全に解消しました。

### 3. 情報量と視認性の両立（ズーム連動の動的テキスト描画）
* **課題**: 知育要素として「開放したマスの地名（例：愛知県）」をマップ上に表示したかったのですが、全国マップをズームアウトした状態ですべての地名を描画すると、文字が重なり合って真っ黒になってしまう問題がありました。
* **解決策**: D3のズーム倍率（`transform.k`）を監視し、それをReactの親要素の CSS Custom Properties（`--zoom-scale`）としてリアルタイムに流し込む設計を採用しました。これにより、SVG内の `<text>` 要素側で `opacity: calc((var(--zoom-scale) - 1.1) * 2.5)` のようなCSSの計算式を用いることができ、「ズームアウト時は透明になり、ズームインするとフワッと地名が浮かび上がる」という、パフォーマンスを落とさず洗練されたインタラクションを実現しました。
